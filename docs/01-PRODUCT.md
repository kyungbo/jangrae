# 처음장례 — 기획 문서 (Product)

> 최종 업데이트: 2026-03-31

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | **처음장례** |
| 도메인 | jangrae.com |
| 핵심 가치 | "당황하지 마세요, 하나씩 안내해 드립니다." |
| 타겟 사용자 | 장례를 처음 치르는 유가족 (30~60대, 정보 비대칭 상태) |
| 제품 유형 | 정보 중심의 매칭 플랫폼 |

### 왜 이 서비스인가?

- 매년 약 30만 건의 장례가 발생하지만, 대부분의 유가족은 **처음** 겪는 상황
- "사망 직후 할 일", "장례식장 비용" 등의 검색량이 높지만 신뢰할 만한 정보가 분산되어 있음
- 상조 업계의 정보 비대칭(불투명한 가격, 불필요한 옵션 유도)이 심각

---

## 2. 사용자 여정 (User Journey)

```
검색 유입 → 콘텐츠 습득 → 도구 활용 → 전문가 연결
```

1. **유입**: "장례식장 비용", "사망 직후 할 일" 등 포털 검색
2. **정보 습득**: 가이드 기사 열람, 체크리스트 확인
3. **도구 활용**: 비용 계산기로 예산 파악
4. **전문가 연결**: 조건에 맞는 장의사/상조 상담 (향후)

| 기능 | 우선순위 | 설명 |
|------|----------|------|

| 사후 행정 가이드 | P2 | 상속, 보험 청구, 명의이전 등 체크리스트화 |

## 5. 수익 모델 (BM)

| 단계 | 모델 | 설명 |

## 7. 디자인 원칙

1. **차분함**: 감정적으로 힘든 상황의 사용자를 위해 절제된 톤
2. **한 번에 하나씩**: 인지 부하를 줄이기 위해 정보를 단계적으로 노출
3. **투명성**: 비용, 절차 등의 정보를 숨기지 않음
4. **즉시 행동 가능**: 모든 정보 하단에 다음 행동(전화, 상담) 유도
